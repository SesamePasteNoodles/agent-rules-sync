# 安全政策

## 回報安全問題

請不要用公開 Issue 回報安全漏洞、可能洩漏的憑證或可造成非預期檔案寫入的
路徑。

請使用 GitHub 儲存庫 `Security` 頁面的 **Report a vulnerability** 功能，
透過 Private Vulnerability Reporting 私下提供：

- 受影響版本或 commit。
- 重現步驟與必要條件。
- 預期與實際行為。
- 可能影響的檔案、權限或資料。
- 已知緩解方式；請勿附上真實憑證或個人資料。

如果 Private Vulnerability Reporting 尚未啟用，請只開一個不含漏洞細節的
Issue，請維護者提供私下聯絡方式。

## 支援範圍

安全修正以最新發布版本與 `main` 分支為主。較舊版本可能要求先升級後再確認
問題是否仍存在。

## 安全邊界

本工具設計為只管理 `config/targets.json` 白名單內的規則檔案，並提供預覽、
備份、暫存寫入與 SHA-256 驗證。以下問題屬於安全回報範圍：

- 能越過白名單讀寫或刪除其他檔案。
- 預覽模式仍修改 Agent 全域目錄。
- 路徑驗證、符號連結或目錄穿越可造成非預期寫入。
- 備份或日誌洩漏認證、Token、Session 或個人資料。
- 回復流程可套用未受管理檔案或在驗證失敗後留下部分變更。

作業系統權限遭入侵、使用者明確手動修改檔案，或使用 `-NoBackup` 後無可用
備份，不屬於工具能完整防護的範圍。
