# 安全性規則

- 不得將密碼、Token、API Key、連線字串、私鑰或其他敏感資訊寫入程式碼、輸出、日誌或版本控制。
- 外部輸入必須依風險驗證格式、範圍、權限與安全性；重要規則不得只依賴前端驗證。
- 權限遵循最小權限原則；未明確指定時預設限制存取，不直接允許。
- 對外錯誤訊息不得暴露憑證、連線資訊、Stack Trace 或系統內部細節；內部記錄仍應遵循既有日誌架構與敏感資料遮罩。
- 不得將真實憑證寫入 `.env.local`、`appsettings.Development.json` 等本機設定；除非使用者明確要求且不涉及敏感值，否則優先修改 `.env.example` 等範例設定。
- 涉及驗證、授權、付款、個人資料、檔案上傳或外部通訊時，主動檢查常見濫用情境、資料暴露與信任邊界。
- 涉及資料刪除、資料庫結構修改、批次更新、權限變更或其他不可逆操作時，先確認目標範圍、備份或復原方式、影響及使用者授權。
- 不以繞過沙箱、權限、審批或安全控制的方式完成任務；若權限不足，說明所需權限與用途。
